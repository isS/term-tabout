需要确认的设计决策

  1. Kill 视觉降权：顶栏与 group footer 的"Kill all"批量按钮已删除（误触代价过高）；行级 ✕ 默认完全隐藏，仅 hover/focus 浮现，且使用 ghost 样式 — 不再用红底强化。批量 kill 仅保留键盘路径：X 进入"删除模式" → 多选 → 确认。
  2. Title 字段（新增）：
    • 默认 = git repo 名 / cwd basename，多层 cwd 时附加 · subdir 后缀
    • 用户按 r 进入 inline 编辑态；⏎ 保存，esc 取消，R 重置回默认
    • 持久化到 ~/.term-tabout/titles.json，按 cwd 索引（PID 易变，不能作为 key）
    • 已重命名的 title 前缀显示 ✎ 暖黄小标记
    • filter 同时匹配 title 与 cmd 字段
  3. 三个时间维度：
    • started — 进程启动绝对时间（今天 HH:MM，跨天加日期）
    • running — 已运行时长 = now − startedAt（绝对值，不衰减）
    • last / idle — 距最近 chpwd / preexec 的相对时间，超阈值（默认 5 min）切换到 idle 标签
    • stale 时 last → died 红色标记
    • 需要在 schema 里新增 startedAt 字段：collector 首次记录时写入，之后只更新 updatedAt
  4. 分组维度：按 git repo 根 / cwd 顶级目录归组。同 cwd 多会话不去重，仅在 group head 标 (2× same cwd) 提示。
  5. 布局：CSS multi-column 瀑布流，4 / 3 / 2 / 1 列在 1180 / 860 / 520 px 切换。
  6. 焦点模型：group 级（蓝边） + row 级（蓝底）。↑↓ 行内移动，←→ 跨 group 切换。
  7. 过滤行为：未命中 group 直接折叠隐藏（密度优先），底部行计提示。
  8. 终端品牌色：iTerm 黄 / Ghostty 紫 / Terminal 灰；cyan + green 留给 Wezterm / Alacritty / Warp。

对 spec 与 plan 的影响

  • Session schema 新增字段： startedAt: number（首次记录时间，毫秒）。title 不入 state 文件。
  • 新增持久化文件： ~/.term-tabout/titles.json — { [cwd]: title }，UI 写、collector 不读。
  • Collector 改动： 首次记录该 PID 时写入 startedAt = now；后续只更新 updatedAt / cwd / lastCmd。
  • 新增按键： r rename · R reset to default · X 批量删除模式。

term-tabout Design Specification

1. Overview

term-tabout 是一个 macOS 系统级的终端 Tab 可视化与管理工具。 它提供类似 tab-out 浏览器扩展的 Bento Grid 仪表盘， 用于在不同终端模拟器（iTerm2、Ghostty 等）之间可视化、管理并跳转活跃的终端会话。

2. Architecture

系统采用解耦架构：基于 Shell 的 Collector + 基于 React Ink 的 UI。

2.1 Collector（Shell 插件）

  • 语言：Zsh / Fish 脚本
  • 钩子：chpwd（目录变更）和 preexec（命令执行前）
  • 功能：采集会话元数据并写入 ~/.term-tabout/states/{PID}.json
  • 采集字段：
    • PID, PPID
    • 当前工作目录（CWD）
    • 终端程序（$TERM_PROGRAM）
    • 最近执行的命令
    • 时间戳

2.2 Storage

  • 活跃会话：~/.term-tabout/states/*.json（每个活跃 shell 一个文件）
  • 持久化书签：~/.term-tabout/saved.yaml（用户保存的路径）

2.3 UI（React Ink）

  • 技术栈：Node.js + React + Ink
  • 布局：响应式 Bento Grid
  • 特性：
    • 通过 fs.watch 实时更新
    • 过期进程检测与清理
    • 按终端区分的视觉品牌（颜色/图标）

3. Interaction Design

4. Implementation Roadmap

  1. Phase 1 — Collector：编写 shell hook 与 state 文件落盘逻辑
  2. Phase 2 — CLI Core：构建 Node.js 骨架与文件 watcher
  3. Phase 3 — UI：使用 Ink 实现 Bento Grid 组件
  4. Phase 4 — Integration：接入 AppleScript 跳转与剪贴板支持
  5. Phase 5 — Refinement：打磨动画、配色与错误处理

Step 2：创建 Zsh Collector
# src/collector/term-tabout.zsh
_term_tabout_record() {
  local state_dir="$HOME/.term-tabout/states"
  mkdir -p "$state_dir"
  local pid=$$
  local cwd=$(pwd)
  local cmd=$1
  local term=$TERM_PROGRAM

  cat <<EOF > "$state_dir/$pid.json"
{
  "pid":  $pid,
  "cwd":  "$cwd",
  "term": "$term",
  "lastCmd":   "${cmd:-"idle"}",
  "updatedAt": $(date +%s)000
}
EOF
}

autoload -Uz add-zsh-hook
add-zsh-hook chpwd _term_tabout_record
preexec_functions+=(_term_tabout_record)

Task 2 — 核心数据引擎 & Watcher

Step 1：定义 Session 类型
import { z } from 'zod';

export const SessionSchema = z.object({
  pid: z.number(),
  cwd: z.string(),
  term: z.string().optional(),
  lastCmd: z.string().optional(),
  updatedAt: z.number()
});

export type Session = z.infer<typeof SessionSchema>;
Step 2：实现 File Watcher
import chokidar from 'chokidar';
import fs from 'fs/promises';
import path from 'path';
import { SessionSchema, Session } from '../types';

export class SessionManager {
  private stateDir = path.join(process.env.HOME!, '.term-tabout/states');

  async getActiveSessions(): Promise<Session[]> {
    const files = await fs.readdir(this.stateDir);
    const sessions = await Promise.all(
      files.map(async (f) => {
        try {
          const content = await fs.readFile(path.join(this.stateDir, f), 'utf-8');
          return SessionSchema.parse(JSON.parse(content));
        } catch { return null; }
      })
    );
    return sessions.filter((s): s is Session => s !== null);
  }

  watch(callback: (sessions: Session[]) => void) {
    const watcher = chokidar.watch(this.stateDir);
    watcher.on('all', async () => {
      const sessions = await this.getActiveSessions();
      callback(sessions);
    });
    return () => watcher.close();
  }
}

Step 2：Dashboard Grid
import React, { useState, useEffect } from 'react';
import { Box, useInput } from 'ink';
import { SessionManager } from '../engine/watcher';
import { Session } from '../types';
import { SessionCard } from './SessionCard';

export const Dashboard = () => {
  const [sessions, setSessions] = useState<Session[]>([]);
  const [focusIndex, setFocusIndex] = useState(0);
  const manager = new SessionManager();

  useEffect(() => manager.watch(setSessions), []);

  useInput((input, key) => {
    if (key.rightArrow) setFocusIndex(i => Math.min(i + 1, sessions.length - 1));
    if (key.leftArrow)  setFocusIndex(i => Math.max(i - 1, 0));
    if (input === 'q')  process.exit();
  });

  return (
    <Box flexDirection="row" flexWrap="wrap">
      {sessions.map((s, i) => (
        <SessionCard key={s.pid} session={s} isFocused={i === focusIndex} />
      ))}
    </Box>
  );
};

import { exec } from 'child_process';

export const teleportToSession = (pid: number, term: string) => {
  if (term === 'iTerm.app') {
    const script = `tell application "iTerm" to activate`; // Simplified
    exec(`osascript -e '${script}'`);
  }
};

export const killSession = (pid: number) => {
  process.kill(pid, 'SIGTERM');
};
